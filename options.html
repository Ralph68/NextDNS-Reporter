Paramètres
Configuration de l'extension NextDNS Reporter.
Fournisseur d'envoi : Formspree
Les signalements sont envoyés via Formspree, un service tiers gratuit. Créez un formulaire, puis copiez l'URL ou l'identifiant ci-dessous. Sans endpoint configuré, les envois sont désactivés.

Accepte l'URL complète ou l'identifiant seul (ex : abcd1234).

Identité par défaut
Ces coordonnées sont pré-remplies automatiquement dans le formulaire. L'email est obligatoire — il permet à l'administrateur de vous répondre. Le prénom est facultatif mais utile pour personnaliser les échanges.
Mode one-click
En mode one-click, un seul clic sur le bouton envoie immédiatement le signalement sans ouvrir le formulaire. Nécessite qu'un email soit mémorisé ci-dessus. Le signalement contient uniquement les informations techniques — pas de contexte ni de commentaire.

Aide
Tout ce qu'il faut savoir pour utiliser NextDNS Reporter efficacement.
🔒
Certificat NextDNS requis
Sans le certificat NextDNS installé dans le navigateur, une page bloquée en HTTPS affiche une erreur SSL standard — pas la page de blocage NextDNS. L'extension ne peut donc pas détecter le blocage ni afficher le bouton.

Ce prérequis n'est pas détectable automatiquement par l'extension. Si le bouton n'apparaît jamais sur les sites bloqués, commencez par installer le certificat :

nextdns.io/certificate →
🔵
Comment signaler un blocage ?
Quand NextDNS bloque un site, une page sombre s'affiche avec le message "[nom du site] est bloqué". L'extension la détecte automatiquement et affiche un bouton bleu centré en bas : Demander le déblocage.

Remplissez votre email (obligatoire), ajoutez un contexte si utile, puis cliquez sur Envoyer la demande. L'administrateur reçoit immédiatement un email.
⚡
Mode one-click
Activable dans Paramètres. En mode one-click, un seul clic sur le bouton envoie directement le signalement — le formulaire ne s'affiche pas.

Nécessite qu'un email soit mémorisé. Le signalement ne contiendra pas de contexte ni de commentaire, mais il sera envoyé instantanément. Idéal si vous signalez souvent.
⏳
Déjà signalé — que faire ?
Pour éviter les doublons, l'extension bloque un nouveau signalement pour le même domaine pendant 48 heures. Le bouton affiche alors "⏳ Déjà signalé".

Si votre demande est urgente, contactez directement l'administrateur par email. L'historique complet est disponible dans l'onglet Historique.
✅
Le site a été débloqué ?
Après envoi, un bouton Rafraîchir la page apparaît dans le formulaire. Cliquez dessus — il vide le cache navigateur et recharge.

Si la page est toujours bloquée après rechargement, essayez Ctrl+Shift+R (Windows) ou Cmd+Shift+R (Mac). Le cache DNS local peut prendre quelques minutes à expirer.
📧
Les emails arrivent dans les spams ?
C'est un comportement courant avec les nouveaux expéditeurs automatiques.

Pour y remédier : ouvrez votre boîte de réception, trouvez le premier email reçu via Formspree, cliquez sur "Ce n'est pas un spam" et ajoutez l'expéditeur à vos contacts. Les emails suivants arriveront en boîte principale.
🔧
Le bouton n'apparaît pas ?
Vérifiez que l'extension est bien activée dans chrome://extensions et que le mode développeur est ON.

Si le bouton était présent avant et a disparu suite à une mise à jour de NextDNS, la détection peut être affectée. Ouvrez F12 → Elements sur la page bloquée et vérifiez que #titleText et #nextdnsLogoGradient existent toujours. Signalez le problème sur GitHub.
🔄
Comment mettre à jour l'extension ?
Remplacez les fichiers dans le dossier source (là où vous l'avez installé), puis ouvrez chrome://extensions et cliquez sur ↺ Recharger sur la carte NextDNS Reporter.

Vos paramètres (email, prénom, endpoint) sont conservés — ils sont stockés séparément dans le navigateur, pas dans les fichiers de l'extension.

Confidentialité
Ce que l'extension fait — et ne fait pas — avec vos données.
Ce que l'extension ne fait jamais
🚫
Aucune collecte sur les pages normales
Le script s'injecte sur toutes les pages par nécessité technique, mais il s'arrête immédiatement à la première ligne si la page n'est pas une page de blocage NextDNS. Il ne lit rien, n'écoute rien, ne transmet rien pendant votre navigation habituelle. Vous pouvez vérifier cela directement dans le code source (fichier content.js, section 1 — Détection).
🚫
Aucun tracking, aucune télémétrie
L'extension ne contient aucun outil d'analyse, de statistiques d'usage, de reporting vers un serveur central, ni aucune dépendance externe. Le code est intégralement en Vanilla JS — aucune librairie tierce n'est chargée.
🚫
Aucun envoi automatique en arrière-plan
Les données ne sont envoyées que lorsque vous cliquez explicitement sur Envoyer la demande. Jamais en arrière-plan, jamais sans votre action.

Ce qui est stocké localement
L'extension stocke ces données dans chrome.storage.local, c'est-à-dire uniquement sur votre appareil. Ces données ne quittent jamais votre navigateur, sauf lors d'un signalement explicite (voir tableau ci-dessus).

⚠️
Les signalements transitent par Formspree
Formspree est un service tiers américain (San Francisco). Les données envoyées lors d'un signalement sont traitées par leurs serveurs avant d'être transmises à l'adresse email de l'administrateur. La politique de confidentialité de Formspree s'applique à ces données : formspree.io/legal/privacy-policy.

Aucune donnée n'est envoyée à Formspree sans votre action explicite.
Vos droits
À tout moment, vous pouvez :

• Consulter l'historique des signalements : onglet Historique ci-dessus.
• Effacer l'historique : bouton "Effacer l'historique" dans l'onglet Historique.
• Modifier ou supprimer l'email et le prénom mémorisés : onglet Paramètres.
• Désinstaller l'extension : supprime automatiquement toutes les données locales.

Pour toute question : github.com/jeantoroot/NextDNS-Reporter/issues
